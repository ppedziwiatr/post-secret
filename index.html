    How it works ↓

    
Encryption happens in your browser. The secret never leaves your device in plaintext. We use AES-256-GCM via the browser's native WebCrypto API — no third-party crypto libraries.


    
The server is dumb. It stores an opaque encrypted blob, returns a random ID, and deletes it on first read. It never sees the key, never sees the plaintext. Even a compromised server leaks nothing useful.


    
The key never touches the server. When using two-channel mode, the decryption key travels separately from the link. In combined mode, the key sits in the #fragment of the URL — browsers never include the fragment in HTTP requests, so no server ever sees it.


    
Hosted on IPFS. The frontend is a single static file pinned to IPFS. Its CID (content hash) is a cryptographic commitment to the exact code you're running — it's impossible to silently modify. You can verify it yourself: ipfs add --only-hash index.html. Traditional web hosting gives you no such guarantee.


    
Zero dependencies. The entire app is one HTML file. The server is 70 lines of Node.js using only built-in modules. Less code, smaller attack surface, easy to audit.
